System Architecture & Design Blueprint

This document outlines the software engineering patterns and security protocols implemented in this FastAPI repository.

Architected by Akhil Jayaraj Python 3.10+ FastAPI 0.110 SQLAlchemy 2.0 Async PyJWT HS256 Bcrypt Encryption

1. Directory Layout & Separation of Concerns

Follows official FastAPI & standard Python project structure guidelines to ensure clean maintainability and scalability:

  • app/core/: Configuration (BaseSettings), database session factory, security primitives.
  • app/models/: SQLAlchemy ORM database models using modern Mapped type annotations.
  • app/schemas/: Pydantic v2 validation models for input serialization and output responses.
  • app/api/v1/: Versioned API endpoint routers and dependency injection utilities (get_current_user).
  • app/templates/ & static/: Responsive glassmorphic frontend UI templates.

2. Authentication & Authorization Flow

  1. Password Hashing: User passwords are encrypted using one-way Bcrypt salt hashes via Passlib.
  2. OAuth2 Password Bearer: Swagger UI and external clients authenticate via /api/v1/auth/login.
  3. Dual Token Security: Short-lived Access Tokens (60 min) for requests and Refresh Tokens (7 days) for session maintenance.
  4. Cookie & Header Interceptors: Native support for both HTTP Bearer headers and HTTP-only cookies.

3. Deployment & DevOps Readiness

This repository is pre-configured for instant automated deployment to cloud platforms:

Containerization (Docker)

docker build -t fastapi-auth-app .
docker run -p 8000:8000 fastapi-auth-app

Render Cloud Deployment (`render.yaml`)

Connect this GitHub repository to Render for automatic zero-config builds using Uvicorn.
